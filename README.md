# Akira MD – Web Pairing (Vercel)

A lightweight web pairing page that lets users connect their WhatsApp to your Akira MD bot by entering their phone number and receiving a pairing code.

---

## 📁 Project Structure

```
akira-web-pairing/
├── api/
│   └── pair.js          ← Serverless function (Baileys pairing logic)
├── public/
│   └── index.html       ← The pairing web page users see
├── vercel.json          ← Vercel routing config
├── package.json
└── README.md
```

---

## 🚀 Deploy to Vercel (Step by Step)

### Option A – Vercel CLI (recommended)

1. Install Vercel CLI:
   ```bash
   npm i -g vercel
   ```

2. From inside this folder:
   ```bash
   vercel
   ```
   Follow the prompts. On first deploy it will ask you to log in.

3. For production:
   ```bash
   vercel --prod
   ```

### Option B – GitHub + Vercel Dashboard

1. Push this folder to a GitHub repository.
2. Go to [vercel.com](https://vercel.com) → **Add New Project**.
3. Import your GitHub repo.
4. Vercel auto-detects the config. Click **Deploy**.
5. Done — you get a live URL like `https://akira-pairing.vercel.app`.

## 🔗 How It Works

1. User visits your Vercel URL (e.g. `https://your-project.vercel.app`)
2. They enter their WhatsApp number **with country code** (digits only, e.g. `2347081827038`)
3. The `/api/pair` serverless function spins up a Baileys socket and requests a pairing code from WhatsApp
4. The code (format: `XXXX-XXXX`) is returned to the page
5. User goes to **WhatsApp → Settings → Linked Devices → Link with phone number** and enters the code
6. Bot is connected ✅

---

## ⚠️ Important Notes

- **Vercel `/tmp` is ephemeral** — session files are deleted after each invocation. This pairing endpoint is for *generating* the code only. Your actual bot (`index.js`) must run on a persistent server (Railway, VPS, Render with always-on, etc.).
- The pairing code is **valid for ~2 minutes** — users must use it quickly.
- **Do not run your full `index.js` bot on Vercel** — WhatsApp bots need a persistent long-running process. Use Vercel only for the web pairing page.

## ✅ Recommended Bot Hosting

For your actual Akira MD bot process, use:
- **Railway** (free tier available, always-on)
- **Render** (free Web Service with `node index.js`)
- **Koyeb**
- **Your own VPS** (DigitalOcean, Hetzner, etc.)

The Vercel pairing page can link to any of these — the session generated by the pairing call connects to whatever server your bot is running on.
